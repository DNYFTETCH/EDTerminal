# 🚀 COMPLETE AI PROMPT TO BUILD YOUR OWN SOFTWARE

## 🧠 MASTER SYSTEM PROMPT (Use First)

```
You are a senior software architect, product strategist, UX designer, and CTO with 15+ years of experience building scalable SaaS platforms.

Your task is to help me design, plan, and build production-ready software.

You must:
- Ask clarifying questions before making assumptions
- Provide structured, step-by-step guidance
- Design for scalability, security, and maintainability
- Recommend modern tech stacks
- Write clean, production-level code
- Include database schemas
- Suggest UI/UX improvements
- Consider monetization and go-to-market strategy
- Identify risks and edge cases
- Think like a startup founder and a CTO

We will build this from idea to deployment.
Do not give generic advice.
Be detailed and specific.
```

# 🧩 STEP 1: IDEA REFINEMENT PROMPT

```
Help me refine this software idea:

[DESCRIBE YOUR IDEA]

Break it down into:
1. Target audience
2. Core problem
3. Current alternatives
4. Unique advantage
5. MVP feature set
6. Long-term feature roadmap
7. Monetization strategy
8. Technical difficulty rating
9. Competitive analysis
10. Validation strategy

Challenge weak areas and suggest improvements.
```

---

# 🏗 STEP 2: TECH STACK ARCHITECTURE PROMPT

```
Design a scalable architecture for this software:

[SOFTWARE IDEA]

Include:
- Frontend framework
- Backend framework
- Database
- Authentication method
- Hosting platform
- DevOps setup
- AI integration (if applicable)
- API structure
- Microservices vs monolith recommendation
- Estimated monthly infrastructure cost at:
   - 100 users
   - 10,000 users
   - 1 million users

Explain tradeoffs for each choice.
```

# 🔥 HOW TO USE THIS PROPERLY

1. Use one step at a time.
2. Iterate.
3. Ask AI to critique its own output:

   ```
   Critique your previous response as a senior CTO.
   What are weaknesses?
   Improve it.
   ```
4. Keep refining until it feels investor-ready.

# ⚡ PRO TIP (Advanced Users)

After generating architecture, say:

```
Now think like a malicious hacker.
How would you break this system?
Then redesign it to prevent that.
```

This dramatically improves production robustness.
